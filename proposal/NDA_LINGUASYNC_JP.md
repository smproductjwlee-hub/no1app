## 前文

[貴社名] (以下「甲」という) と [当社名] (以下「乙」という) は、SaaS 型サービス **「LinguaSync」** の導入に関する検討・協議・契約交渉 (以下「本目的」) を行うにあたり、相互に開示する秘密情報の取扱いについて、以下のとおり契約を締結する。

> **本契約の位置づけ**
> 本 NDA は SOW (作業範囲記述書) および DPA (データ処理委託契約書) の締結に先立つ、検討段階での秘密情報保護を目的とする。本契約締結後に SOW / DPA を別途締結した場合、それらの守秘義務条項が並存し、より厳格な方が適用される。

## 1. 目的 (Purpose)

本契約は、本目的の遂行に必要な範囲で、当事者が相互に秘密情報を開示することを前提に、当該情報の保護のために必要な事項を定めることを目的とする。

### 1.1 本目的の範囲
- LinguaSync サービスの機能・仕様・価格に関する協議
- 導入規模・運用体制の検討
- パイロット運用 / PoC (実施する場合)
- SOW / DPA / 利用契約の交渉

### 1.2 本目的外の利用禁止
当事者は、本目的以外で秘密情報を使用してはならない。特に、自社製品・サービスの企画開発・改良・販売活動に流用する行為は本目的外とみなす。

---

## 2. 秘密情報の定義 (Definition of Confidential Information)

### 2.1 秘密情報とは
本契約で「秘密情報」とは、当事者の一方 (以下「開示者」) から他方 (以下「受領者」) に対し、本目的のために開示・提供される情報のうち、以下のいずれかに該当するものをいう。

#### A. 形態
| 種別 | 例 |
|---|---|
| 書面・電子データ | 提案書・見積書・仕様書・契約書草案・スプレッドシート |
| 口頭 | 会議・電話・面談での発言 |
| 視覚情報 | デモ画面・プロトタイプ・施設見学 |
| 物理媒体 | 試作品・サンプル・USB 等 |

#### B. 内容 (例示・限定列挙ではない)
**甲側の秘密情報の例**
- 店舗運営ノウハウ・スタッフ構成・人事情報
- 売上・経営数字・取引先情報
- 既存システムの構成・課題
- 個人データ (スタッフ氏名等)

**乙側の秘密情報の例**
- LinguaSync のアーキテクチャ・ソースコード・API 仕様
- 価格戦略・コスト構造・利益率
- 他顧客の情報 (匿名化の有無を問わず)
- 開発ロードマップ・未公開機能
- 取引先・業務委託先情報

### 2.2 秘密情報の特定

#### 2.2.1 書面・電子データ
- 「秘」「Confidential」「社外秘」等の表示があるもの、または
- 文脈上明らかに秘密性を有するもの

#### 2.2.2 口頭・視覚情報
- 開示時に「これは秘密情報である」旨を明示
- 開示後 14 日以内に書面 (メール可) で要旨を確認することが望ましい (確認漏れがあっても、客観的に秘密性が明らかな場合は秘密情報として扱う)

#### 2.2.3 個人データ
個人情報 (個人情報保護法上の) は、表示の有無にかかわらず常に秘密情報として扱う。

## 3. 秘密情報からの除外 (Exceptions)

以下のいずれかに該当する情報は、秘密情報に含まれない。受領者がこれを主張する場合、立証責任は受領者が負う。

| # | 除外事由 |
|---|---|
| ① | 開示時点で既に公知であった情報 |
| ② | 開示後、受領者の責によらず公知となった情報 |
| ③ | 開示時点で受領者が既に正当に保有していた情報 |
| ④ | 受領者が秘密情報を使用せず独自に開発した情報 |
| ⑤ | 守秘義務を負わない第三者から正当に入手した情報 |

## 4. 秘密情報の取扱い (Handling)

### 4.1 守秘義務
受領者は秘密情報を、本目的のためにのみ使用し、開示者の事前書面同意なく第三者に開示・漏えいしてはならない。

### 4.2 善管注意義務
受領者は、自己の同種の情報に対するのと同等以上の注意 (少なくとも善良なる管理者の注意) をもって秘密情報を管理する。

### 4.3 開示できる範囲
受領者は、以下の者にのみ秘密情報を開示できる:

| 開示先 | 条件 |
|---|---|
| 自社の役職員 | 本目的の遂行に必要な者に限定 / 本契約と同等以上の守秘義務を課す |
| 業務委託先・代理人 (顧問弁護士・税理士・監査法人等) | 本目的の遂行に必要 / 守秘義務契約を締結済み |
| 親会社・子会社 | 同等の守秘義務 + 開示者への事前通知 |

その他の第三者 (フランチャイズ加盟店・取引先・顧客等) への開示は、開示者の **事前書面同意** が必要。

### 4.4 複製・改変
- 受領者は本目的の遂行に必要な範囲でのみ秘密情報を複製・改変できる
- 複製物にも秘密情報の表示を維持する
- 不要な複製は行わない

### 4.5 安全管理
- 書類: 施錠可能な場所に保管
- 電子データ: アクセス制御 / 暗号化通信 / 業務関係者のみ閲覧可
- 持ち出し: 必要最小限・記録を残す
- 廃棄: シュレッダー / 安全消去ツール

## 5. 法令上の開示 (Compelled Disclosure)

### 5.1 開示が必要となる場合
受領者が以下の事由により秘密情報を開示する義務を負う場合、本契約違反とはならない:
- 法令の規定
- 裁判所・捜査機関・監督官庁等の命令
- 上場規則等に基づく開示義務

### 5.2 通知義務
受領者は、上記による開示の前に (法令上禁止される場合を除き)、速やかに開示者に通知し、開示範囲・方法について協議する。開示者が異議を申し立てる場合、合理的な範囲で協力する。

---

## 6. 知的財産権の不譲渡

- 本契約に基づく秘密情報の開示は、開示者から受領者への知的財産権 (著作権・特許権・商標権・営業秘密等) の譲渡・実施許諾を一切意味しない
- 受領者は秘密情報を基に、開示者の事前書面同意なく特許出願・知的財産権の取得を行わない

---

## 7. 反映·派生著作物の取扱い

### 7.1 受領者が秘密情報を基に作成した文書・図表・分析等
- 「派生情報」として、当該情報も秘密情報と同様に扱う
- 本目的終了後は §9 に従い返還・破棄

### 7.2 共同作業の成果物
- 本目的の検討過程で双方の知見を統合した成果物 (議事録・PoC レポート等) は、双方の秘密情報を含むものとし、相互に守秘義務を負う

## 8. 期間 (Term)

### 8.1 契約期間
- 本契約の有効期間: 締結日から **2 年間**
- 期間満了 30 日前までに双方から書面の終了通知がない場合、1 年間自動更新

### 8.2 早期終了
- 当事者は 30 日前の書面通知により本契約を解約できる
- ただし、終了時点までに開示済みの秘密情報については、§10 の残存期間中は守秘義務が継続する

### 8.3 SOW / DPA 締結による吸収
- 本契約期間中に SOW / DPA が締結された場合、本目的に関する守秘義務はそれらの契約に承継される
- 本契約は SOW / DPA 終了まで効力を維持し、矛盾がある場合は SOW / DPA が優先する

---

## 9. 終了時の取扱い

本契約の終了 (期間満了・解約) 時、または開示者からの書面要請があった場合、受領者は秘密情報を以下のいずれかにより処理する。

### 9.1 受領者の選択肢

**選択 A: 返還**
- すべての秘密情報 (複製物・派生情報を含む) を開示者に返還する
- 期限: 終了通知から **30 日以内**

**選択 B: 破棄**
- すべての秘密情報を安全に破棄する (シュレッダー・電子的安全消去等)
- 期限: 終了通知から **30 日以内**
- 開示者の請求があれば、破棄完了報告書を発行する

### 9.2 例外
以下は返還・破棄の対象外:
- 法令上の保存義務がある情報 (税務関係書類等)
- 自動バックアップシステム上に技術的に残存する情報 (ただし、§10 の守秘義務は継続)
- 議事録・契約交渉記録のうち、後の紛争対応に必要なもの

本契約終了後も、以下の条項は **5 年間** 効力を有する:

| 条項 | 内容 |
|---|---|
| §4 | 秘密情報の取扱い (守秘義務) |
| §6 | 知的財産権の不譲渡 |
| §10 | 残存条項 (本条) |
| §11 | 損害賠償 |
| §13 | 紛争解決 |

ただし、開示時点で営業秘密に該当する情報は、当該情報が公知化するまで秘密性を維持する (5 年経過後も守秘義務継続)。

## 11. 違反·損害賠償

### 11.1 違反時の措置
受領者が本契約に違反した場合、開示者は以下を行うことができる:
- 違反行為の即時停止請求 (差止請求)
- 損害賠償請求
- 本契約の解除

### 11.2 損害賠償
- 受領者は、違反により開示者に生じた損害 (合理的な弁護士費用を含む) を賠償する
- **損害賠償額の上限**: 本 NDA 単独の場合、特段の定めなし (実損害ベース)
- ただし、本 NDA と並行して SOW を締結した場合、SOW §16 の上限規定 (直近 12 ヶ月の受領額) を準用するものとする

### 11.3 差止請求
- 秘密情報の漏えい・不正使用は受領者に生じた損害が金銭賠償だけでは不十分であることを双方確認する
- 開示者は、損害賠償に加え、または替えて、差止請求権を行使できる
- 仮処分・本案訴訟いずれの方法も可

## 12. 一般条項

### 12.1 譲渡禁止
当事者は、相手方の事前書面同意なく、本契約上の地位・権利・義務を第三者に譲渡してはならない。

### 12.2 完全合意 (Entire Agreement)
本契約は、本目的に関する当事者間の完全な合意を構成し、本契約締結前の一切の口頭・書面合意に優先する。

### 12.3 修正
本契約の修正は、双方代表者の記名押印 (または電子署名) ある書面でのみ可能。

### 12.4 一部無効
本契約のいずれかの条項が無効・執行不能とされた場合も、他の条項の効力には影響しない。当事者は当該無効条項を可能な限り近い意味の有効条項に置き換える。

### 12.5 通知
本契約に基づく通知は、以下の方法で行う:
- 書面 (郵送): 各当事者の本店所在地宛 / 配達証明郵便
- 電子メール: §13 記載のアドレス宛
- 受領確認: 郵送は配達日 / メールは送信から 24 時間で到達したものとみなす

## 사용 가이드 (영업 흐름상의 위치)

```
[1단계] 첫 미팅 / 관심 표명
        ↓
[2단계] NDA 체결  ← 이 문서 ⭐
        ↓
[3단계] 상세 제안서·견적 (PROPOSAL + PRICING) 송부
        ↓
[4단계] 보안 검토 (SECURITY.md) / 데이터 검토
        ↓
[5단계] SOW + DPA 동시 체결
        ↓
[6단계] MVP 개발 착수 (¥5M 착수금)
        ↓
[7단계] M3 본격 가동 / 月額 청구 시작
```

### NDA 체결 타이밍 팁
- **PoC 또는 데모 전**: 「데모를 보여주려면 NDA 가 먼저 필요합니다」 라고 자연스럽게 요청하면 거부감 없음
- **상세 견적·고객 데이터 공유 전**: 고객사가 점포·스탭 정보를 공유하기 전에 체결되어야 함
- **PoC 데이터 사용시**: 고객의 실제 운영 데이터로 PoC 를 돌릴 경우 필수

### 「먼저 NDA 가 필요합니다」 라고 말하는 타이밍의 효과
1. 프로다움 어필 — 「제대로 일을 하는 회사」 라는 인상
2. 정보 보호 의식 어필 — 보안 검토 단계에서 점수가 됨
3. 일본 대기업 대상에는 거의 필수 (NDA 없이 미팅을 진행하지 않는 회사 다수)

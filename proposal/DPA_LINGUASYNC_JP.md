## 前文

[貴社名] (以下「委託者」または「甲」という) と [当社名] (以下「受託者」または「乙」という) は、LinguaSync サービス (以下「本サービス」) の提供に伴い、甲が乙に取扱いを委託する個人データに関する事項を本契約で定める。

本契約は SOW (作業範囲記述書) および本サービス利用規約と併せて適用され、矛盾がある場合は本 DPA が個人データの取扱いに関する事項について優先する。

## 1. 用語定義

| 用語 | 定義 |
|---|---|
| 個人データ | 個人情報保護法第 16 条に定める「個人データ」(本契約上は GDPR 第 4 条「Personal Data」を含む) |
| データ主体 | 個人データにより識別される個人 (本サービスでは主にスタッフ) |
| 処理 | 個人データの収集・記録・保管・編集・利用・提供・削除等のすべての行為 |
| 委託者 (甲) | 個人データ取扱事業者 (Controller) として本サービスを利用する [貴社名] |
| 受託者 (乙) | 個人データ処理者 (Processor) として委託者から処理を委託される [当社名] |
| 再委託先 (サブプロセッサ) | 受託者が個人データの処理の一部を再委託する第三者 (例: Render, Supabase, Google Cloud Translation) |
| 漏えい等 | 個人データの漏えい・滅失・毀損・不正アクセス・不正利用 |
| 監督官庁 | 日本: 個人情報保護委員会 (PPC) / EU: 各国データ保護当局 |

## 2. 委託の内容

### 2.1 処理の対象・性質・目的

| 項目 | 内容 |
|---|---|
| 処理の対象 | 委託者の店舗スタッフおよび店舗管理者の個人データ |
| 処理の性質 | 本サービスの提供に必要な収集・保管・表示・翻訳・削除 |
| 処理の目的 | LinguaSync サービスを通じた多言語コミュニケーション支援 |
| 処理の期間 | 本契約の有効期間中、および §11 に定める終了後の取扱い期間 |

### 2.2 取扱い個人データの種類

#### A. スタッフ (主たるデータ主体)
| 項目 | 必須 / 任意 | 用途 |
|---|---|---|
| 氏名 (表示名) | 必須 | 画面表示・指示の宛先 |
| ログイン ID (メールまたはハンドル) | 必須 | 認証 |
| パスワードハッシュ (bcrypt) | 必須 | 認証 |
| 言語設定 | 必須 | 翻訳出力先選択 |
| プロフィール写真 (アバター) | 任意 | 画面表示 |
| 所属店舗 (ワークスペース ID) | 必須 | データ分離 |
| 役職・グループ | 任意 | 通知範囲設定 |
| 応答ログ (指示への回答内容) | 必須 | 業務記録 |
| ログイン履歴 (IP・タイムスタンプ) | 必須 | セキュリティ監査 |

#### B. 管理者 (店舗マネージャー)
| 項目 | 必須 / 任意 | 用途 |
|---|---|---|
| 氏名 | 必須 | 画面表示 |
| メールアドレス | 必須 | 認証・通知 |
| パスワードハッシュ | 必須 | 認証 |
| 操作ログ | 必須 | 監査 |

#### C. 取扱わないデータ
- マイナンバー (個人番号)
- クレジットカード番号 / 銀行口座番号 (決済代行を通じて取得しない)
- 医療・健康情報
- 思想・信条・宗教
- 人種・民族
- 犯罪歴
- その他「要配慮個人情報」(個人情報保護法第 2 条第 3 項)

> **重要**: 委託者は要配慮個人情報を本サービスに入力してはならない。違反による損害は委託者の責任とする。

## 3. 受託者の義務 (Processor Obligations)

### 3.1 一般義務
- 受託者は、委託者の文書による指示に従い個人データを処理する
- 本契約で許可された目的以外で個人データを処理しない
- 法令上の義務がある場合を除き、個人データを第三者に開示しない

### 3.2 安全管理措置 (Security Measures)
別紙 1「安全管理措置一覧」に従う。要旨:

| 区分 | 措置 |
|---|---|
| 物理的 | クラウドデータセンタ (Supabase Tokyo / Render) の物理セキュリティに依拠 |
| 技術的 | TLS 1.2+ 暗号化通信 / bcrypt パスワードハッシュ / JWT セッション / WAL モード DB / 自動バックアップ |
| 組織的 | アクセス権限の最小化 / 操作ログ記録 / 従業者の機密保持義務 |
| 人的 | 従業者への教育・研修 / 退職時のアクセス権剥奪 |

詳細は別紙 1 および `SECURITY.md` を参照。

### 3.3 従業者の監督
- 受託者は個人データに接触する従業者・委託先に対し、本契約と同等の義務を課す
- 機密保持契約を締結し、業務必要範囲のみアクセスを許可する

### 3.4 再委託 (Sub-processing)
受託者は、別紙 2「承認済み再委託先一覧」に記載の事業者に処理を再委託することができる。

#### 3.4.1 現行の再委託先 (別紙 2 抜粋)
| サービス | 提供事業者 | 役割 | 所在地 |
|---|---|---|---|
| アプリケーションホスティング | Render Services, Inc. | サーバ稼働 | 米国 (オレゴン州) ※将来日本リージョンへ移行検討 |
| データベース | Supabase, Inc. | DB ホスティング | **日本 (東京リージョン)** |
| 機械翻訳 | Google LLC (Cloud Translation API) | 翻訳処理 | 日本リージョン優先 |
| メール送信 | [SendGrid 等] | 通知メール送信 | 米国 |

#### 3.4.2 再委託先の追加・変更
- 新たな再委託先を追加する場合、受託者は **30 日前まで** に委託者に書面通知する
- 委託者は通知から 14 日以内に異議を申し立てることができる
- 異議があった場合、当事者は誠実に協議する

#### 3.4.3 再委託先の管理
- 受託者は再委託先に対し、本契約と同等の安全管理措置を課す
- 再委託先の違反は受託者の違反とみなす

### 3.5 データ主体の権利対応
データ主体から開示・訂正・削除等の請求を受けた場合:

| 段階 | 主体 | 期限 |
|---|---|---|
| 請求の一次窓口 | 委託者 (甲) | データ主体への対応責任者 |
| 受託者への協力依頼 | 委託者 → 受託者 | 必要時 |
| 受託者の対応 | 委託者の指示に従い対応 | 営業日 7 日以内 |

受託者は委託者の指示なくデータ主体に直接回答しない (法令上の義務がある場合を除く)。

### 3.6 漏えい等発生時の通知 ⭐
- 受託者は漏えい等を認知した時点から **72 時間以内** に委託者に通知する
- 通知内容:
  - 概要 (発生日時・覚知経緯)
  - 影響を受けたデータ主体の概数・データ種類
  - 影響範囲・原因 (判明している範囲)
  - 既に講じた対応・今後の対応予定
  - 受託者の連絡窓口
- 監督官庁・データ主体への通知義務は委託者に帰属するが、受託者は資料提供等で協力する

### 3.7 個人情報保護委員会 (PPC) 等への対応
- 監督官庁から本サービスに関する報告徴収・立入検査等の通知を受けた場合、受託者は速やかに委託者に通知する
- 委託者と協議のうえ対応する

### 3.8 記録の保持
受託者は以下の記録を 5 年間保持する:
- 処理活動の記録 (Article 30 GDPR 相当)
- アクセスログ
- 漏えい等発生時の対応記録

- 利用料の期日内支払い

## 5. 国外移転 (International Data Transfer)

### 5.1 原則
- スタッフ・管理者の個人データは原則として **日本国内 (Supabase 東京リージョン)** で保管・処理する

### 5.2 例外
以下のケースで一時的に日本国外のシステムを経由することがある:
- 翻訳 API (Google Cloud Translation): 日本リージョン優先で利用するが、API 仕様により他リージョンを経由する場合がある
- アプリケーションホスティング (Render): 現状米国オレゴン (将来日本リージョン移行を検討)
- メール送信サービス: 米国経由

### 5.3 移転時の保護措置
- すべての通信は TLS 1.2+ で暗号化
- 各再委託先と適切なデータ処理契約 (DPA) を締結
- GDPR 適用範囲の場合、SCC (Standard Contractual Clauses) または十分性認定に依拠

### 5.4 EU/EEA 居住者データ
- EU/EEA 居住者の個人データを本サービスで処理する場合、GDPR Chapter V (国際移転) 規定を遵守する
- 委託者は事前に受託者に通知すること

## 6. 監査 (Audit)

### 6.1 委託者の監査権
- 委託者は年 1 回を限度として、受託者の本契約遵守状況について監査を実施できる
- 監査は事前 30 日通知のうえ、営業日に実施する
- 監査費用は委託者負担。ただし、重大な違反が発見された場合は受託者負担

### 6.2 監査の代替
受託者は以下の証憑提供により、立入監査に代えることができる:
- 第三者監査報告書 (SOC 2 / ISO 27001 等を取得した場合)
- セキュリティポリシー (`SECURITY.md`) の最新版
- 過去 12 ヶ月のインシデント報告書 (有無)

### 6.3 監査範囲の制限
- 他の顧客の個人データに影響する範囲は監査対象外
- 受託者の機密情報・知的財産に該当する技術詳細は除外可

## 7. 委託期間中の処理

### 7.1 アクセス制御
- 受託者の従業者で個人データにアクセスできるのは「業務上必要な者」に限定
- アクセス権付与・剥奪はチケット管理し、年 2 回見直す

### 7.2 暗号化
- 通信: TLS 1.2 以上必須
- 保管: パスワードは bcrypt (cost factor ≥ 12)
- バックアップ: AES-256 暗号化

### 7.3 バックアップ
- 日次自動バックアップ (Supabase 標準機能)
- 保持期間: 30 日
- 復元テスト: 半年に 1 回

### 7.4 ログ記録
| ログ種別 | 保持期間 | 用途 |
|---|---|---|
| アクセスログ | 12 ヶ月 | セキュリティ監査 |
| 操作ログ | 24 ヶ月 | 不正アクセス検知 |
| エラーログ | 6 ヶ月 | トラブルシュート |

## 8. 終了時の取扱い (End of Processing) ⭐

### 8.1 委託者の選択権
本契約終了時、委託者は以下のいずれかを選択する:

**選択 A: データ返還**
- 委託者は終了日から **30 日以内** に CSV / JSON 形式で全データをエクスポート可能
- エクスポート機能はスーパー管理者画面より提供 (操作マニュアル参照)

**選択 B: データ削除**
- 受託者は終了通知から **90 日以内** に全個人データを安全に削除する
- バックアップを含むすべての媒体から削除
- 削除完了後、委託者に「削除完了報告書」を発行

### 8.2 削除の例外
以下は削除義務の対象外とする:
- 法令上の保存義務がある記録 (請求書・契約書等)
- 匿名化処理されたうえで統計目的に変換されたデータ
- 紛争対応上必要な記録 (係争終結まで)

### 8.3 削除完了報告書の記載事項
- 削除日時
- 削除対象データの範囲
- 削除実施者・確認者
- 削除方法 (DB 削除・バックアップ削除・媒体破棄)

## 9. 損害賠償 (Liability)

### 9.1 受託者の責任
- 本契約違反により委託者または第三者に損害を与えた場合、受託者は損害を賠償する
- 賠償額は SOW §16 の上限規定 (直近 12 ヶ月の受領額) に従う
- ただし、個人情報の漏えい等により委託者がデータ主体・監督官庁に対し法令上の責任を負う費用 (通知費用・行政罰金・訴訟費用等) については、受託者の故意・重過失による場合に限り、上限を超えて負担することがある

### 9.2 第三者請求への対応
- データ主体・第三者から請求を受けた当事者は、相手方に速やかに通知する
- 当事者は協力して対応を協議する
- 各自の責任範囲に応じて費用を分担する

### 9.3 過失割合
- 委託者・受託者の双方に過失がある場合、過失割合に応じて分担
- 委託者が要配慮個人情報を入力した等、委託者起因の漏えいは委託者の責任
